How to get it running locally

  • Downloaded Visual Stuido Web Express
  • Open the Solution "Lol.Api.Challenge2.sln"
  • Compile
  • Create a Directory for storing you data - for example "c:\temp"
  • Enter this directory in the config files of all projects
    • \Lol.Matchloader.Console\app.config
    • \Lol.Bilgewater.Melter\app.config
    • \Lol.Bilgewater\web.config
    Look for following Element: <appsettings> <add key="DataPath" value="[your path]" /> </appsettings>
  • Create two subdirectories
    • \BILGEWATER\Source
    • \RANKED_SOLO\Source
  • Download the Datasets provided for the challenge and extract the contents to the folders. No Subfolders, all the JSON files have to be in the "Source" folders
  • Now you have to download the matchdata by running the \Lol.Matchloader.Console\bin\debug\Lol.Matchloader.Console.exe [region] [source] [apikey]
    • region is one of the known regions like na, euw, lan, ...
    • source is BILGEWATER or RANKED_SOLO
    • apikey is your apikey for accessing riot api
  • Do this for all combinations of region and source
  • Copy the mercs.xml form \Lol.Bilgewater\Content to [your data store]\metadata
  • Run \Lol.Bilgewater.Melter\bin\debug\Lol.Bilgewater.Melter.exe
  • Place you apikey in the \Lol.Bilgewater\web.config Look for following Element: <appsettings> <add key="ApiKey" value="[your key]" /> </appsettings>
  • When your data is prepared you can launch the Webapp \Lol.Bilgewater

Notes

  • Modified Javascript files are
    • Scripts\Chart.js for X-Axis Icon display
    • Scripts\Site.js for all Script-logic
